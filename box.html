h1태그는 화면 전체를 사용, a태그는 줄바꿈이 되지않고 다른 콘텐츠들과 같은 라인에 위치

이런 차별은 왜 존재하는가?
h1태그는 제목태그이므로 화면 전체를 사용하는 것이 더 편리하고, 링크는 화면 전체를 사용하는 것이 더 불편해서 딱 자기의 컨텐츠 크기만큼을 쓰는 것이라고 짐작할 수 있음
 > 즉 태그의 성격과 일반적인 쓰임에 따라 달라짐

화면 전체를 쓰는 태그를 블럭 레벨 엘리먼트, 자신의 컨텐츠 크기만큼을 쓰는 것을 인라인 엘리먼트라고 부름
블럭 레벨 엘리먼트인 태그의 스타일에 display:inline; 속성을 추가하면 자신의 컨텐츠 크기만큼 차지하게 만들 수 있음
즉 언제든 바꿀 수 있음
display:none;이라고 하면 아예 태그의 내용이 화면에서 사라진다

padding : 컨텐츠 내부의 여백
margin : 컨텐츠 외부의 여백
width 값을 변경하면 100px의 크기(폭)만큼 태그의 크기가 변화, 높이를 변경하고 싶다면 height 값을 변경하면 된다

style 태그 내 겹치는 내용이 많다면 aa, bb{}로 해서 공통되는 부분을 묶는 것도 가능하다.

border-color, border-style, border-width 등
